Project Outcomes:

The outcomes we are aiming to achieve for our project is ensuring that the minimal viable features are successfully implemented. The features that will be demonstrated consist of Quizzes, Interactive Modules, Achievement System, Anonymous Message and Pop-Up Feature.

In order for the Quizzes Feature to be deemed as successful it will require 10 different questions to be generated to the user based on the module they have been learning. Furthermore, ensures that the user has answered the set amount of questions correctly in order to earn a certificate that showcases they have successfully completed the module.

This takes us to the next feature which is essentially the main part of our website known as the Interactive Modules. For the Interactive Modules to be successfully implemented onto the website it needs to have the following:

  • Correct information displayed on each type of the modules.
  • Activities where the user answers questions based off a video they have watched, a dilemma system where users are required to select the correct dilemma which ensures they are understanding the information before taking the quiz
  • Pop-Up System to ensure any difficult words for the user are defined so they can understand what they are learning.

For the Achievement System to be correctly implemented it needs to ensure that the users name and module completed has successfully been displayed on the certificate that we are providing users with.

Furthermore, the Anonymous Message Box can be deemed as complete if an icon is displayed which enables the user to be able to type a message on any queries they may have regarding the SafeMe website. However it should also appear on the users screen about what they have asked as well as the answer that was given by one of our members.

In order for all outcomes to be complete the Pop-Up Feature needs to be implemented onto the website. This will be achieved if the user is able to hover over a underlined word and a box will appear on the screen where the user is able to get further information, definition or a fact regarding the module.

Narrator Function

This extended feature will be used in written areas such as articles and modules. The Narrator Function is an optional module where the user is able to ‘tab’ select an icon next to a section of texts, which will enable a narrator to read the text that is being provided to users. The purpose behind this feature is to provide those with handicaps an opportunity to learn what they are passionate about and not feel left out.

Progress

Testing

Throughout our project we had used a variety of testing techniques such as Systems Testing, Smoke Testing and Component Testing. To ensure that the quiz feature was working properly we would use Smoke Testing to ensure that when the user had answered the correct amount of questions it would allow them to progress to the next module and if they had not answered enough questions right the user will be required to redo the test till they have successfully reached the passing mark, with the use of the Smoke Test we were able to ensure that the quiz was able to successfully be implemented onto our project.

Furthermore Component Testing was achieved to ensure that the right pages would load when a user had selected that link to a different page as well as ensuring that the correct modules were displayed when users had decided what module that wanted to start learning. This was also complimented with the use of System Testing as we were able to identify that some of our team members had younger siblings that were in the right age that we were aiming to appeal our websites to, which enabled us to ensure that the design chosen by the group would appeal to those 10 years old and above. Throughout this time of testing some of the design was changed in an attempt to make it professional looking but with the ability to engage those 10 years or above with the use of more interactivity on our website as well as the images selected. If we were given more time the group would have been able to incorporate the other techniques of testing such as Unit Testing and Integration Testing, however with the testing we have been able to incorporate it ensured we were able to create a fully functioning website that would appeal to our audience.
